
Para cumplir con la exigencia de evaluar el impacto de la STL en el rendimiento, optamos por el Camino 2 (Experimento Complementario). Desarrollamos un benchmark para comparar la inserción de 10 millones de elementos en un std::vector usando push_back, con y sin el uso previo de reserve(). Observamos empíricamente que el uso de reserve() eliminó las reasignaciones dinámicas de memoria y las costosas copias de elementos, logrando que el programa fuera aproximadamente [X] veces más rápido. Esto demuestra que conocer el funcionamiento interno de la memoria contigua en la STL es crucial para optimizar el costo temporal en C++.
